🗣 Issue 1 — Missing cart review page
"I wanted to see all my items before confirming the order. It went straight to confirmation." — Arjun

"I didn't see the burger icon at the top. I was just scrolling down." — Meena

Section 6
Design Improvement Suggestions
Actionable recommendations to improve the FoodieGo app based on usability findings.
🛒
Add a Cart Review Screen
Insert a dedicated cart page between Food Detail and Order Confirmation so users can review items, quantities, and total before placing the order. This reduces accidental orders.
High Priority
🏷️
Label Category Section Clearly
Add a visible "Browse by category" heading above the category chips on the home screen. Increase chip size and add subtle animation to draw attention for first-time users.
High Priority
👆
Increase Touch Target Sizes
Enlarge the + and − quantity buttons on the Food Detail screen to at least 44×44px as per Apple HIG and Android guidelines. This especially helps older users and those with motor difficulties.
Medium Priority
🗺️
Add Live Map Tracking
Integrate a live map view on the Order Tracking screen showing the delivery rider's real-time location. This is now a standard expectation set by Swiggy and Zomato.
Medium Priority
✅
Improve Tracking Step Visibility
Make the active tracking step more visually prominent — use a pulsing animation or a larger icon. Add a clear label like "Current status" so all users can instantly identify progress.
Medium Priority
🖼️
Add Image Gallery on Food Detail
Allow users to swipe through multiple food images on the detail screen. Add small dot indicators below the image to signal that it is swipeable.
Low Priority

📊 Conclusion

The usability testing of FoodieGo revealed that the core flow — browsing, selecting, and ordering food — is largely intuitive for tech-savvy users. However, first-time and older users faced challenges with navigation discoverability, small touch targets, and a missing cart review step. By implementing the 6 recommendations above — particularly adding a cart page and improving category visibility — the app's usability score can be significantly improved. Iterative testing after each design change is recommended to validate improvements.
